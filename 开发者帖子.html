🚧 遇到的挑战

  • Prompt 工程精度：让 Gemini 生成「不机器、有人味」的中文文案非常困难。经过 50+ 次迭代调试，为每种文案类型设计了专用 prompt 模板，包括角色设定、语气控制、格式约束等。
  • API 调用稳定性：Gemini API 偶尔返回空响应或超时。实现了前端重试逻辑和友好的错误提示，确保用户体验不中断。
  • 隐私与成本平衡：不想自建后端存储用户数据（成本+隐私风险），最终采用「用户自带 API Key + 浏览器本地存储」的方案，零服务器成本运行。
  • 单文件全功能：把完整的 SaaS 产品压缩到一个 HTML 文件，包含模板选择、参数配置、结果展示、复制分享等功能，在保持代码可维护性的同时做到功能完整。
